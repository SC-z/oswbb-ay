# 模块 Spec

## 1. 模块概览
- 模块名称：meminfo
- 所属子系统：日志解析层
- 架构位置：解析与内存指标提供
- 一句话职责描述：解析 meminfo 日志并提供内存与 Swap 指标趋势。

## 2. 设计背景与问题域
- 背景说明：meminfo 为键值型输出，字段数量多且采样频繁。
- 需要解决的问题：稳定抽取关键内存指标并保持结构化。
- 约束条件：字段来自文本键名，需与内核输出保持一致。

## 3. 设计目标与非目标
- 设计目标：
  - 解析关键内存字段并构建时间序列。
  - 提供基础趋势查询给上层分析。
- 非目标：
  - 不负责异常阈值策略与报告渲染。

## 4. 核心职责与边界
- 核心职责：解析日志，维护 `MemStats` 与时间序列访问方法。
- 职责边界说明：仅做解析与数据提供。
- 不应承担的责任：输出格式化、跨日志合并与时间过滤策略。

## 5. 架构位置与协作关系
- 上游模块：`processor`。
- 下游模块：`common`（趋势结构）。
- 依赖方向说明：解析结果被 `processor` 与 `output` 使用。

## 6. 核心设计与关键流程
- 抽象设计说明：通过时间戳行切分快照，逐行解析 key/value。
- 关键流程描述：
  - 识别 `zzz ***` 行 → 生成快照。
  - 解析键值对 → 更新 `MemStats`。
  - 形成 `MemInfoLog` 时间序列。

## 7. 关键实现说明
- 核心接口 / 数据结构：`MemInfoParser.ParseFile`、`MemInfoLog`、`MemStats`。
- 实现要点：
  - 仅解析已列举字段，未知字段忽略。
  - 时间戳统一转为 CST。
- 与设计意图的映射：以稳定解析为主，避免噪声字段影响。

## 8. 设计取舍与已知限制
- 设计取舍：未覆盖所有 meminfo 字段。
- 已知限制与风险：
  - 新字段需显式加入解析表。
  - 时间范围与异常检测逻辑在 `processor`，非本模块。

## 9. 演进方向与重构提示
- 可能的演进方向：
  - 通过配置驱动字段映射，减少硬编码。
  - 增加字段完整性检测。
- 修改注意事项：保持字段名称与内核输出一致。
