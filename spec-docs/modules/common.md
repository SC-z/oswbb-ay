# 模块 Spec

## 1. 模块概览
- 模块名称：common
- 所属子系统：基础工具层
- 架构位置：通用结构与统计工具
- 一句话职责描述：提供时间序列结构与基础统计方法。

## 2. 设计背景与问题域
- 背景说明：多个解析模块需要共用时间序列与异常结构。
- 需要解决的问题：避免重复定义基础数据结构。
- 约束条件：保持轻量、无业务依赖。

## 3. 设计目标与非目标
- 设计目标：
  - 统一 `TimeValue` 与 `TrendAnomaly` 结构。
  - 提供基础统计方法（Average/Max/Min）。
- 非目标：
  - 不承载业务指标或复杂统计算法。

## 4. 核心职责与边界
- 核心职责：基础类型定义、简单统计函数。
- 职责边界说明：不依赖其他模块。
- 不应承担的责任：解析、输出、业务逻辑。

## 5. 架构位置与协作关系
- 上游模块：`iostat`、`meminfo`、`processor`。
- 下游模块：无。
- 依赖方向说明：仅被依赖，不反向依赖。

## 6. 核心设计与关键流程
- 抽象设计说明：值对象与小型工具函数。
- 关键流程描述：无流程，仅数据结构与统计方法。

## 7. 关键实现说明
- 核心接口 / 数据结构：`TimeValue`、`TimeValueList`、`TrendAnomaly`。
- 实现要点：统计函数为纯计算，避免副作用。
- 与设计意图的映射：为上层提供一致的时间序列语义。

## 8. 设计取舍与已知限制
- 设计取舍：功能极简，复杂统计放在上层实现。
- 已知限制与风险：统计函数未处理 NaN/Inf。

## 9. 演进方向与重构提示
- 可能的演进方向：增加稳健统计工具（MAD/IQR）基础实现。
- 修改注意事项：保持无依赖与轻量特性。
